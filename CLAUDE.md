# CLAUDE.md — News Fact-Checking Assistant

Контекст проекта для продолжения работы в Claude Code. Портфолио-проект
(ML/NLP-инженерия) для заявок на CS/AI программы за рубежом.

## Что строим

Систему проверки достоверности новостей с двумя точками входа:

- **Telegram-бот** — пересылаешь ему новость (текст или ссылку), получаешь разбор.
- **Браузерное расширение** (Manifest V3) — берёт текст открытой страницы, шлёт на бэкенд, показывает результат.

Обе точки входа обращаются к общему бэкенду на **FastAPI**.

## Главный принцип дизайна

Это **НЕ «детектор правды»**, выдающий процент достоверности. LLM не может
надёжно измерить «насколько новость правдива», а фейковый процент опаснее
отсутствия оценки — создаёт illusion of precision.

Это **ассистент для проверки**, который:

- раскладывает новость на атомарные проверяемые утверждения (claims);
- по каждому claim ищет источники, показывает их с датами и типом (первоисточник / перепечатка / мнение);
- по каждой паре (claim, источник) определяет позицию: supports / refutes / not enough info;
- проверяет **независимость** источников (20 перепечаток одного пресс-релиза = одно доказательство, не двадцать);
- флагит манипулятивные признаки (кликбейт-заголовок ≠ тексту, анонимные источники, эмоциональная лексика, отсутствие дат/имён);
- **честно говорит «не могу проверить»**, когда данных нет.

## Архитектура пайплайна

```
Точки входа (TG-бот + расширение)
        ↓
Извлечение текста (trafilatura, обход пейволла/cloudflare)
        ↓
Разбиение на утверждения (LLM → атомарные claims)
        ↓
Поиск доказательств (веб-поиск + rerank, по каждому claim отдельно)
        ↓
Проверка независимости (near-duplicate дедуп по эмбеддингам, поиск первоисточника)
        ↓
Определение позиции (stance: supports / refutes / not enough info, по каждой паре claim×источник)
        ↓
Вердикт + объяснение (по claim'ам, с флагами манипуляций, с цитированием источников)
```

По сути это RAG наизнанку: не «найти ответ», а «найти доказательства за и против утверждения».

### Инженерные заметки

- **Проверка независимости** — самый ценный для портфолио шаг. Кластеризация near-duplicate по эмбеддингам + попытка дотянуться до первоисточника. Здесь демонстрируется понимание задачи, а не «прикрутил LLM».
- **Stance** делать по каждому claim к каждому источнику отдельно, НЕ одним общим промптом на всю новость. Точнее + прозрачнее объяснение.
- Свежие новости требуют веб-поиска в реальном времени (модель без поиска не знает событий после cutoff).

## Стек

- **FastAPI** — бэкенд (уже знаком по RAG-чатботу).
- **trafilatura / newspaper3k / readability** — извлечение текста статьи из URL.
- **pgvector** — кэш доказательств, эмбеддинги для дедупликации (уже знаком).
- **Qwen2.5** — LLM для claim extraction, stance, генерации объяснений. Мультиязычен, русский тянет прилично.
- Веб-поиск + reranking для retrieval доказательств.
- Docker + CI (по образцу предыдущих проектов).
- Обучение/эксперименты — Google Colab (нет локального GPU).

## Датасеты

Расклад асимметричный: для английского задача обеспечена данными, для русского — почти нет.

### Английский (зрелая экосистема)

- **AVeriTeC** (Schlichtkrull et al., 2023) — ПРИОРИТЕТ. Реальные claims + веб-доказательства + вердикты + обоснования (структура QA). Повторяет наш пайплайн один-в-один. Основной бенчмарк.
- **FEVER / FEVEROUS** — claims против Википедии, метки supports/refutes/NEI. Для обучения и оценки stance-компонента.
- **LIAR** — короткие политические claims с градацией достоверности (PolitiFact).
- **MultiFC** — крупный набор реальных claims с множества фактчекинговых сайтов.
- **CheckThat! Lab (CLEF)** — ежегодные соревнования (check-worthiness, фактчекинг, claim normalization, retrieval), мультиязычные, живые.

### Русский (мало данных, политически нагружены)

- Kuzmin et al. (2020) — детекция фейков для русского. Маленькие наборы: 174 текста (Pisarevskaya 2017) + 8867 текстов (1366 фейков / 7501 реальных). Бинарно.
- **Fakespeak-RUS** (2024–2025) — новее, метки от независимого фактчекера (несут отпечаток конкретного агентства).
- Казахско-русский корпус (2025) — структурно самый полезный: аннотация из 7 категорий (CLAIM, SOURCE, EVIDENCE, DISINFORMATION_TECHNIQUE, AUTHOR_INTENT, TARGET_AUDIENCE, TIMESTAMP). Ровно наши под-задачи.

**Подводный камень:** фактчекинг на русском политически нагружен, метки несут bias. Для академической заявки — не проблема, а плюс, если честно отрефлексировать в README/отчёте.

## Стратегия по данным

- Ядро пайплайна **строить и оценивать на английском** (AVeriTeC + FEVER для stance) — есть на чём мерить метрики.
- Русский — как **демонстрация cross-lingual переноса**: Qwen2.5 мультиязычен, показываем работу на low-resource языке. Маленькие русские наборы — для качественной оценки, НЕ для обучения с нуля.
- Academically звучит сильнее, чем «обучил классификатор фейков»: демонстрируются retrieval, stance detection, калибровка неопределённости, осознанная работа с bias.

## Открытые вопросы / следующие шаги

1. **MVP-скоуп по этапам** — минимальный сквозной прототип на паре claim'ов (какой шаг делать первым).
2. **Метрики** — как измерять качество stance-компонента; как честно считать «уверенность» вердикта без фейкового процента (калибровка).
3. Определиться со схемой независимости источников (порог near-duplicate, эвристики первоисточника).
